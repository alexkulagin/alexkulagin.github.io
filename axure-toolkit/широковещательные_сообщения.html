Содержание

javascript:


     /* функция обработчик */

     function MyHandler (channel, message) {

          console.log(message);

     }


     /* регистрация обработчика */

     $m.listen('channel-name', ['widgetA', 'widgetB', MyHandler]);


void 0;

Регистрирует слушателя для сообщений канала "channel.name". В качестве обработчиков слушателя выступают два виджета и одна функция. В время обработки сообщения в виджетах с именами "widgetA" и "widgetB" будет вызвано событие "OnMove", после вызова будет выполнен последний обработчик — функция "MyHandler".

Пример регистрации списка обработчиков

javascript:


     /* функция обработчик */

     function MyHandler (channel, message) {

          console.log('канал:', channel);

          console.log('сообщение:', message);

     }


     /* регистрация обработчика */

     $m.listen('channel-name', MyHandler);


void 0;

Регистрирует слушателя для сообщений канала "channel.name". В качестве обработчика слушателя выступает функция "MyHandler".

Пример регистрации функции обработчика

javascript:

     $m.listen('channel.name', 'widget.name');

void 0;

Регистрирует слушателя для сообщений канала "channel.name". В данном примере виджет с названием "widget.name" будет обработчиком слушателя. В время обработки сообщения в виджете "widget.name" будет вызвано событие "OnMove".

Сценарии для взаимодействия с широковещательными сообщениями.

Кейсы взаимодействия

channel — обязательный параметр с названием канала, который будет прослушиваться. Параметр "channel" может быть строкой или массивом строк (может прослушивать несколько каналов). Если в качестве значения "channel" передан символ "*", то происходит прослушивание всех каналов.


listener — обязательный параметр слушателя. Параметр "listener" может быть строкой, функцией или списком строк и функций. В виде строк передаются имена виджетов, в момент срабатывания на этих виджетах автоматически запускается событие "OnMove".


once — не обязательный параметр, который гарантирует, что слушатель сработает только один раз.

$m.listen(channel, listener, once);

Регистрирует слушателя для сообщений конкретного канала или целого списка каналов.

Метод listen

channel — обязательный параметр с названием канала, на котороый адресуется сообщение. Параметр "channel" может быть строкой или массивом строк.


message — не обязательный параметр сообщения. Параметр "message" может быть любого типа — строка, число, массив или объект.

$m.send(channel, message);

Отправляет сообщение на конкретный канал или список каналов.

Метод send

К интерфейсу широковещательных сообщений можно обратиться через глобальный объект "$m", который содержит два публичных метода, один для отправки сообщений и второй для регистрации слушателей. Слушателями могут быть виджеты, функции или массив, состоящий из виджетов и функций.

Расширение Axure Toolkit обеспечивает передачу широковещательных сообщений, которые можно адресовать в любое место прототипа – в динамическую панель, в местер, в любой фрейм, где бы он не находился.


Широковещательные сообщения или по другому пользовательские сообытия — это то, чего очень сильно не хватает в Axure RP 8. С помощью широковещательных сообщений можно создавать по-настоящему независимые компоненты.


Второй момент — во время демонстрации локальной версии прототипа, в таких браузерах как "chrome" или "firefox", попытки вызова функций в родительских или дочерних фреймах блокируются правилом ограничения домена (origin policy).


Axure Toolkit снимает это ограничение и позволяет с помощью широковещательных сообщений управлять дочерними или родительскими фреймами.
