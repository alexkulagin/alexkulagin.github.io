Список методов

Если менять данные "Repeater" средствами прототипа Axure, то обновление будет происходить для каждого элемента. Например, если одним действием обновить параметр в таблице из 10 строк, то за одно изменение HTML представление будет обновлено 10 раз, если действий 2, то "Repeater" обновится 20 раз и т.д.

Дыра в производительности Axure

javascript:

    

     /* находит виджет по названию */

     var widget = $a('@' + 'Название виджета');


     /* возвращает контроллер для виджета Repeater */

     var controller = widget.getPanelController();


     /* выполняются манипуляции с данными модели */

     ....


     /* применение изменений и отрисовка HTML представления */

     controller.render();


void 0;

Ключевой метод контроллера — после внесения изменений в модель виджета "Repeater", метод "render" применяет изменения и инициирует обновление HTML представления.

Метод render

Модель виджета "Repeater" — это обычный массив JavaScript. Каждый элемент этого массива является объектом конкретной строки. В свойствах объекта строки перечислены названия и значения столбцов (объектное представление таблицы).

Что из себя представляет модель

/* Пример модели, которая состоит из 3 строк и двух столбцов name и size */

[

     { name: "Model A", size: "120" },

     { name: "Model B", size: "100" },

     { name: "Model C", size: "80" }

]

Как было указанно ранее — контроллер с помощью специальных методов осуществляет различные манипуляции над данными из клонированной модели виджета "Repeater".
